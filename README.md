# 大学東方合同合宿 企画ナビ README

## 概要
このフォルダは、大学東方合同合宿の自由参加企画を参加者向けに見やすく表示するための公開用ファイル一式です。

現在の構成では、企画情報は Google スプレッドシートで管理し、Google Apps Script 経由で JSON として取得し、HTML 側で表示しています。

---

## このフォルダに入っているもの
- `index.html`  
  通常ページです。センシティブでない企画を表示します。

- `sensitive.html`  
  センシティブ企画用ページです。通常ページ内の隠し導線からアクセスする想定です。

- `README.md`  
  このファイルです。運用ルールと引き継ぎ用メモをまとめています。

---

## 基本構成
- **企画データ管理**: Google スプレッドシート
- **データ出力**: Google Apps Script
- **表示ページ**: `index.html` / `sensitive.html`
- **公開先**: Cloudflare Pages などの静的ホスティング

---

## 更新方法
### 1. 企画の追加・修正
Google スプレッドシートの `events` シートを編集してください。  
通常の運用では、HTML を直接触る必要はありません。

### 2. 表示の変更
デザインや見た目、文言、表示順などを変えたい場合は `index.html` または `sensitive.html` を編集してください。

### 3. 公開内容の反映
- スプレッドシートの内容変更は、Apps Script 経由で表示ページに反映されます。
- HTML を変更した場合は、Cloudflare Pages 側へ再アップロードしてください。

---

## `events` シートの列
`events` シートは以下の列構成を前提としています。

```text
title,date,start_time,end_time,location,summary,host,category,sensitive
```

### 各列の意味
- `title`  
  企画名

- `date`  
  開催日

- `start_time`  
  開始時刻

- `end_time`  
  終了時刻。未定の場合は `未定`

- `location`  
  開催場所

- `summary`  
  企画の簡単な説明

- `host`  
  主催者名

- `category`  
  企画カテゴリ

- `sensitive`  
  センシティブ判定。`TRUE` ならセンシティブ、`FALSE` なら通常企画

---

## 入力ルール
### 日付
`YYYY/MM/DD` 形式で入力します。  
例: `2026/08/20`

### 時刻
`HH:MM` 形式で入力します。  
例: `20:00`

深夜 0 時をまたぐ場合は、終了時刻に `0:00` を入力してください。  
システム側で翌日 0:00 として扱います。

終了時刻が不明な場合は `未定` と入力してください。

### 主催者名
同じ人が複数企画を持つ場合は、表記を統一してください。

### カテゴリ
表記ゆれを減らすため、なるべく固定の名称を使ってください。  
例:
- 対戦
- 大会
- 作業
- 鑑賞
- 交流
- 原作
- その他

### センシティブ判定
- `FALSE` : 通常ページに表示
- `TRUE` : センシティブ用ページに表示

判定に迷う場合は、実行委員内で確認してください。

---

## 表示ルール
### 通常ページ `index.html`
- `sensitive = FALSE` の企画だけ表示します。
- 現在開催中の企画を最上部に表示します。
- 30分以内に開始する企画をその次に表示します。
- そのほかの企画は日時順で表示します。

### センシティブページ `sensitive.html`
- `sensitive = TRUE` の企画を表示します。

---

## 注意事項
- `events` シートの**列名は変更しないでください**。
- Apps Script の URL を HTML から消さないでください。
- HTML 側は見た目の変更用です。通常運用ではシート更新だけで十分です。
- センシティブページは隠し導線を置いているだけなので、厳密なアクセス制限ではありません。
- `sensitive.html` の存在を知っている人はアクセスできるため、必要以上に秘匿情報は書かないでください。

## よく触る場所

## 引き継ぎ向けメモ
この仕組みは、**企画データの更新をスプレッドシートだけで完結させる**ことを重視しています。  
そのため、通常の企画追加や修正では HTML や Apps Script を触らなくて大丈夫です。

大きく仕様を変えたい場合だけ、HTML 側や Apps Script 側を編集してください。
